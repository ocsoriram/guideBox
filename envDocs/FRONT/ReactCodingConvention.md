## Reactのコーディング規約まとめ

1. **公式規約は存在しない**
    - Reactはライブラリなので、AngularやVueのような厳密な公式コーディング規約はない。
2. **実質的な標準**
    - コミュニティで広く使われている規約やツールが「デファクトスタンダード」になっている。
    - 代表例：
        - **Airbnb JavaScript Style Guide**
        - **ESLint + eslint-plugin-react**
        - **Prettier**
3. **よくあるルール**
    - コンポーネント名はPascalCase、propsはcamelCase。
    - 1ファイル1コンポーネントを基本とする。
    - JSXは読みやすいように条件分岐を工夫する。
    - Hooksは必ずコンポーネントの先頭で呼び出す。
    - 状態(state)は必要最低限にする。
4. **実務での対応**
    - **ESLint + Prettierで自動整形**し、Airbnbガイドラインをベースにする。
    - チーム独自のルール（命名規則、フォルダ構造など）を追加で決めると良い。

---

要するに、**「公式の規約はないが、Airbnbガイドライン＋ESLint＋Prettierが事実上の標準」** です。
